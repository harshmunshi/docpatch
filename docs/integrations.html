01 / Content

No file required.

parse_text() turns any string — LLM output, a streamed buffer, an API response — into an addressable DocTree without touching disk.

A

In-memory strings

The existing open_doc(path) reads a file. parse_text() takes a raw string — no path, no filesystem. The rest of the pipeline is identical.

python 
from docpatch import parse_text, edit_doc
from docpatch.parsers.markdown import serialize
from docpatch.models.anthropic import AnthropicClient

model = AnthropicClient(model="claude-haiku-4-5-20251001")

# 1. Generate a draft — content lives only in memory
draft = model.complete(
    "Write a one-page design doc for a rate-limiter service.",
    max_tokens=2048,
).text

# 2. Parse into an addressable tree — same as open_doc(), no file
tree = parse_text(draft)

# 3. Locate the target node and patch only that section
patched = edit_doc(
    tree,
    "Make the Failure Modes section more concise.",
    model=model,
)

# 4. Back to a string — or write to disk if you want
result: str = serialize(patched).decode()
Key property

Token cost is flat regardless of document length. The LLM receives only the target node + a breadcrumb — never the whole draft. A 20-page design doc costs the same input tokens as a 1-page draft for a targeted edit.

B

Streamed byte output

Buffer the streamed tokens as they arrive, then parse the complete buffer into a DocTree when the stream closes. Patching happens after streaming — the two stages are fully decoupled.

python 
import anthropic
from docpatch import parse_text, edit_doc
from docpatch.models.anthropic import AnthropicClient

client = anthropic.Anthropic()
buffer: list[str] = []

# 1. Stream generation — buffer chunks as they arrive
with client.messages.stream(
    model="claude-haiku-4-5-20251001",
    max_tokens=2048,
    messages=[{"role": "user", "content": "Write a rate-limiter design doc."}],
) as stream:
    for text in stream.text_stream:
        buffer.append(text)

# 2. Stream closed — parse the complete text into a DocTree
draft = "".join(buffer)
tree = parse_text(draft)

# 3. Patch surgically — only one section goes back to the model
model_client = AnthropicClient()
patched = edit_doc(tree, "Add a Redis failover note to the Storage section.", model=model_client)

python 
from docpatch import parse_text, edit_doc

def parse_stream(token_iter, model, instruction: str) -> str:
    """Buffer any token iterator, parse when done, patch surgically."""
    draft = "".join(token_iter)       # drain the iterator
    tree = parse_text(draft)
    patched = edit_doc(tree, instruction, model=model)

    from docpatch.parsers.markdown import serialize
    return serialize(patched).decode()

# Works with any iterator: httpx SSE, websocket frames, local generator, etc.
result = parse_stream(my_token_stream, my_model_client, "Add an example.")
C

Format detection on strings

parse_text() defaults to Markdown. Pass fmt= explicitly when the format is known, or use detect_format() with the MIME type.

python 
from docpatch import parse_text
from docpatch.core.types import DocFormat

# explicit format
tree = parse_text(json_string, fmt=DocFormat.JSON)
tree = parse_text(markdown_string, fmt=DocFormat.MARKDOWN)

# detect from content-type header (e.g. from an HTTP response)
from docpatch.parsers import detect_format_from_mime
fmt = detect_format_from_mime("application/json")   # → DocFormat.JSON
tree = parse_text(body, fmt=fmt)
02 / Patterns

Four patterns for agent loops.

DocPatch's immutable DocTree and deterministic splicer compose cleanly with the most common agentic control-flow shapes.

Pattern 01

Generate → Verify → Patch

Generate a draft, validate each section structurally, send only failing nodes back to the model for repair. The validator sees the full DocTree; the LLM sees only the failing subtree.

Pattern 02

Atomic multi-patch

Apply multiple independent edits to different nodes in a single splice() call. All patches reference the original tree — ordering is irrelevant, conflicts surface at splice time.

Pattern 03

Route by node type

Use a cheap model for prose edits, a powerful model for code blocks. The routing decision happens after locate() — a single function before apply().

Pattern 04

Batch across documents

DocTree objects are immutable; edit_doc() is stateless. Fan out across a file glob with ThreadPoolExecutor — no locks, no shared state.

python 
from docpatch import parse_text, edit_doc
from docpatch.parsers.markdown import serialize
from docpatch.core.tree import DocTree
from dataclasses import dataclass

@dataclass
class Issue:
    fix_instruction: str   # what to tell the model

def check_examples(tree: DocTree) -> list[Issue]:
    """Return an Issue for every section that has no code block child."""
    issues = []
    from docpatch.core.types import NodeType
    for node in tree.walk():
        if node.type == NodeType.HEADING:
            has_code = any(c.type == NodeType.CODE_BLOCK for c in node.children)
            if not has_code:
                issues.append(Issue(
                    fix_instruction=f"Add a concrete code example under '{node.content}'"
                ))
    return issues

def generate_and_refine(prompt: str, model, validators: list) -> str:
    # 1. Generate the initial draft
    draft = model.complete(prompt, max_tokens=3000).text
    tree = parse_text(draft)

    # 2. Validate → patch only the failing nodes — full doc never re-sent
    for validator in validators:
        for issue in validator(tree):
            tree = edit_doc(tree, issue.fix_instruction, model=model)

    return serialize(tree).decode()

result = generate_and_refine(
    "Write a design doc for a rate-limiter.",
    model=my_model_client,
    validators=[check_examples],
)
Why this works

The validator receives the full DocTree — structural checks, word counts, heading hierarchy. The LLM fix call receives only the failing subtree. A 50-section doc with 3 failing sections costs 3× one-section input tokens, not 50×.

python 
from docpatch.parsers.markdown import parse, serialize
from docpatch.locator import CompositeLocator
from docpatch.patcher.replace import ReplaceOperation
from docpatch.splicer import splice
from pathlib import Path

data = Path("spec.md").read_bytes()
tree = parse(data)

model = my_model_client
locator = CompositeLocator(model=model, threshold=0.6)
patcher = ReplaceOperation(max_retry=3)

# Build patches independently — all reference the *original* tree
edits = [
    "Tighten the introduction",
    "Add a Redis failover note to the Storage section",
    "Shorten the Alternatives section to bullet points",
]

patches = []
for instruction in edits:
    result = locator.locate(tree, instruction)
    if result.node_ids:
        patch = patcher.apply(tree, result.node_ids[0], instruction, model)
        patches.append(patch)

# Apply atomically — ordering irrelevant, conflicts raise SpliceError
new_tree = splice(tree, patches)
Path("spec.md").write_bytes(serialize(new_tree))
Atomicity guarantee

All patches reference node IDs from the original tree. splice() applies them in one pass — the document is never in a half-edited state. If two patches target the same node, SpliceError.CONFLICT is raised before any write occurs.

python 
from docpatch.core.types import NodeType
from docpatch.models.openai import OpenAIClient
from docpatch.models.anthropic import AnthropicClient
from docpatch.locator import CompositeLocator
from docpatch.patcher.replace import ReplaceOperation
from docpatch.splicer import splice

# Two models — different cost/quality trade-offs
prose_model = OpenAIClient(model="gpt-4o-mini")     # fast, cheap
code_model  = AnthropicClient(model="claude-sonnet-4-6")  # strong reasoning

def route(tree, node_id) -> object:
    node = tree.get(node_id)
    if node and node.type == NodeType.CODE_BLOCK:
        return code_model
    return prose_model

# Locate with either model (cheap model is fine for location)
locator = CompositeLocator(model=prose_model)
patcher = ReplaceOperation()

result = locator.locate(tree, instruction)
if result.node_ids:
    model = route(tree, result.node_ids[0])  # choose model after locate
    patch = patcher.apply(tree, result.node_ids[0], instruction, model)
    new_tree = splice(tree, [patch])
Cost model

Location is cheap: the locator only sees the content skeleton (headings, code block previews), not full body text. Route the expensive model only to nodes where the quality difference matters — typically code, tables, and structured JSON.

python 
from concurrent.futures import ThreadPoolExecutor, as_completed
from pathlib import Path
from docpatch import open_doc, edit_doc
from docpatch.parsers.markdown import serialize
from docpatch.models.openai import OpenAIClient
import traceback

# Stateless — safe to share across threads
model = OpenAIClient(model="gpt-4o-mini")
INSTRUCTION = "Ensure every section has a concrete code example."

def process(p: Path) -> tuple[Path, str]:
    tree = open_doc(p)
    patched = edit_doc(tree, INSTRUCTION, model=model)
    p.write_bytes(serialize(patched))
    return p, "ok"

files = list(Path("docs/").glob("**/*.md"))

with ThreadPoolExecutor(max_workers=8) as pool:
    futures = {pool.submit(process, p): p for p in files}
    for fut in as_completed(futures):
        p = futures[fut]
        try:
            _, status = fut.result()
            print(f"✓ {p.name}")
        except Exception:
            print(f"✗ {p.name}"); traceback.print_exc()
Why it is safe

DocTree is an immutable Pydantic model. edit_doc() returns a new tree — it never mutates the input. OpenAIClient holds no mutable state. All three facts together mean ThreadPoolExecutor needs no synchronization.

03 / Frameworks

One seam. Every framework.

Adapting a new framework is one class, one method. The ModelClient protocol is the only bridge DocPatch needs.

ModelClient protocol — implement this to connect any LLM
python 
from docpatch.models.base import ModelClient, ModelResponse
from typing import Protocol, runtime_checkable

@runtime_checkable
class ModelClient(Protocol):
    def complete(self, prompt: str, *, max_tokens: int) -> ModelResponse: ...
python — adapter 
from langchain_core.language_models import BaseChatModel
from langchain_core.messages import HumanMessage
from docpatch.models.base import ModelResponse

class LangChainModelClient:
    """Adapt any LangChain BaseChatModel to DocPatch's ModelClient protocol."""

    def __init__(self, llm: BaseChatModel) -> None:
        self._llm = llm

    def complete(self, prompt: str, *, max_tokens: int) -> ModelResponse:
        msg = self._llm.invoke(
            [HumanMessage(content=prompt)],
            config={"max_tokens": max_tokens},
        )
        text = msg.content if isinstance(msg.content, str) else str(msg.content)
        usage = getattr(msg, "usage_metadata", None)
        return ModelResponse(
            text=text,
            tokens_in=getattr(usage, "input_tokens", 0),
            tokens_out=getattr(usage, "output_tokens", 0),
            model_id=getattr(self._llm, "model_name", ""),
        )
python — tool 
from langchain_core.tools import tool
from docpatch import open_doc, edit_doc
from docpatch.parsers import get_parser, detect_format
from pathlib import Path

def make_docpatch_tool(llm: BaseChatModel):
    client = LangChainModelClient(llm)

    @tool
    def edit_document(file_path: str, instruction: str) -> str:
        """Edit a specific section of a document.
        Surgical patching — only the relevant node is sent to the model.
        Args: file_path (Markdown or JSON), instruction (what to change and where)."""
        p = Path(file_path)
        tree = open_doc(p)
        patched = edit_doc(tree, instruction, model=client)
        parser = get_parser(detect_format(p, p.read_bytes()))
        p.write_bytes(parser.serialize(patched))
        return f"Edited '{p.name}'"

    return edit_document

04 / Advanced

Ground-up pipeline control.

The protocols are runtime_checkable. Implement Locator or ModelClient yourself — DocPatch's splicer and storage work with any conforming object.

A

Full manual pipeline

Every stage is individually importable and composable. The convenience API (edit_doc) is a thin wrapper over these exact calls.

python 
from pathlib import Path
from docpatch.parsers.markdown import parse, serialize
from docpatch.locator import CompositeLocator
from docpatch.patcher.replace import ReplaceOperation
from docpatch.splicer import splice
from docpatch.models.openai import OpenAIClient

model   = OpenAIClient(model="gpt-4o-mini")
locator = CompositeLocator(model=model, threshold=0.6)
patcher = ReplaceOperation(max_retry=3)

data = Path("spec.md").read_bytes()
tree = parse(data)                                      # 1. parse

result = locator.locate(tree, "Tighten the Failure Modes section")
if result.confidence < 0.6:                              # 2. locate
    raise ValueError(f"Low confidence: {result.candidates}")

patch    = patcher.apply(tree, result.node_ids[0],     # 3. patch
               "Tighten the Failure Modes section", model)

new_tree = splice(tree, [patch])                         # 4. splice
Path("spec.md").write_bytes(serialize(new_tree))         # 5. serialize

print(f"tokens in={patch.tokens_in} out={patch.tokens_out}")
B

Custom Locator

Bypass the LLM for location when the node ID is already known — e.g. returned by a vector search or a previous DocPatchLoader retrieval.

python 
from docpatch.locator.base import LocateResult
from docpatch.core.tree import DocTree

class DirectLocator:
    """Treat instruction as a literal node ID — zero model calls."""

    def locate(self, tree: DocTree, instruction: str) -> LocateResult:
        if instruction in tree:
            return LocateResult(node_ids=[instruction], confidence=1.0, method="direct")
        return LocateResult(node_ids=[], confidence=0.0, method="direct")

class VectorLocator:
    """Use a vector DB query result (which carries node_id) to skip the LLM."""

    def __init__(self, vector_db) -> None:
        self._db = vector_db

    def locate(self, tree: DocTree, instruction: str) -> LocateResult:
        results = self._db.query(embed(instruction), limit=3)
        ids = [r["node_id"] for r in results if r["node_id"] in tree]
        return LocateResult(
            node_ids=ids[:1],
            confidence=results[0]["score"] if ids else 0.0,
            candidates=ids[1:],
            method="vector",
        )
C

Custom ModelClient

Wrap any completion API — a local model, a proxy, a batch API — in the single complete() method. No base class to inherit, no registration required.

python 
import requests
from docpatch.models.base import ModelResponse

class OllamaClient:
    """Call a local Ollama instance — no API key, fully offline."""

    def __init__(self, model: str = "llama3.2", base_url: str = "http://localhost:11434") -> None:
        self._model = model
        self._url = f"{base_url}/api/generate"

    def complete(self, prompt: str, *, max_tokens: int) -> ModelResponse:
        resp = requests.post(self._url, json={
            "model": self._model, "prompt": prompt,
            "stream": False, "options": {"num_predict": max_tokens},
        }, timeout=120)
        resp.raise_for_status()
        data = resp.json()
        return ModelResponse(
            text=data["response"],
            tokens_in=data.get("prompt_eval_count", 0),
            tokens_out=data.get("eval_count", 0),
            model_id=self._model,
        )

# Works identically with all DocPatch operations
from docpatch import open_doc, edit_doc
tree = open_doc("spec.md")
patched = edit_doc(tree, "Tighten the intro.", model=OllamaClient())
